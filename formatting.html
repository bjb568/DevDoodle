Formatting Help

Our markdown-inspired formatting parser allows you to add formatting to your post such as emphasis and code.

Inline formatting

Inline formatting (formatting that won't cause a line break) is written as a formatting sequence followed by the affected text followed by the same formatting sequence.

Emphasis

Use an asterisk to *emphasise* text that is slightly more important or is stressed in speech.
Use an asterisk to emphasise text that is slightly more important or is stressed in speech.

Strong Emphasis

Use an underscore to add _strong emphasis_ to text that is more important.
Use an underscore to add strong emphasis to text that is more important.

Deleted Text

Use 3 dashes to indicate ---deleted text--- which was once, but is no longer relevant, useful, or accurate.
Use 3 dashes to indicate deleted text which was once, but is no longer relevant, useful, or accurate.

Inserted Text

Use 3 plusses to indicate ---inserted text--- which was added later to the post, especially if it replaced deleted text.
Use 3 plusses to indicate inserted text which was added later to the post, especially if it replaced deleted text.

Inline Code (not parsed)

To indicate `code`, enclose it in backticks.
To indicate code, enclose it in backticks.

Links

Bare URLs with http:// or https:// are automatically turned into a link:

http://example.com
example.com

But it is preferable to use a link title:

[Example title](http://example.com)
Example title

Nesting

You can combine multiple formatting commands, placing one inside another. This is called nesting.

---deleted text with *emphasis* inside---
deleted text with emphasis inside

Escapement

To write literal control characters, for example the literal string "3*4*5 = 60", escape the parsed characters with a backslash (3\*4\*5 = 60). To write a literal backslash, escape it with another backslash.

*emphasis* \*just asterisks\*
emphasis *just asterisks*

Super/sub-script

For convienece, a superscripted string consisting of just word characters (alphanumeric or underscore (_)) can be written with just a carrot.

5^2 = 25
52 = 25

Otherwise parenthatheses are needed.

3^(3^(3)) = 7625597484987
333 = 7625597484987

A dollar sign ($) is used for subscript.

a$(n) = 4n
an = 4n

Block formatting

Block formatting is the control of the type of block a line is placed in. By default, every line is put into a paragraph block.

Blockquote

Blockquotes can contain any material a post can.

> Blockquote > > Blockquote inside a blockquote

Blockquote

Blockquote inside a blockquote

Unordered list

Unordered lists items are started with a dash (- ) or an asterisk (* ). Additional lines inside a list item must be indented by 4 spaces or a tab character.

* Item 1 - Item 2 Line 2 of item 2
  • Item 1

  • Item 2

    Line 2 of item 2

Ordered list

Ordered lists are like unordered lists, except that items have a defined order and are usually rendered with numbers prefixed to list items. Items are started with a sequence of one or more digits (0-9) or a single letter (A-Z, case insensitive) followed by either ) or . (period character). Additional lines inside a list item must be indented by 4 spaces or a tab character.

1. Item 1 b. Item 2 Line 2 of item 2 C) Item 3 39. Item 4
  1. Item 1

  2. Item 2

    Line 2 of item 2

  3. Item 3
  4. Item 4

Block code (not parsed)

Lines indeneted by 4 spaces or a tab character (or more depending on nesting in lists) are not parsed and are put in a code block.

function multiply3(a, b, c) { return a * b * c; }
function multiply3(a, b, c) { return a * b * c; }
