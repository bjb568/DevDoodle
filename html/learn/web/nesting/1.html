Nesting

The Document Object Model

Each tag that we write gets downloaded and interpreted by the browser, which converts them into a tree structure called the DOM (Document Object Model). You can see the tree structure by opening the web inspector in your browser.

In an HTML file you write tags, but the DOM represents relationships between elements. An element will often have:

  • A parent
  • A list of children
  • Siblings

These relationships depend on which tags are nested within other tags. A child is nested in a parent if the start tag for the child is after the start tag of the parent and the close tag of the child is before the close tag of the parent: <parent><child></child></parent>.

To continue, add a tag nested within another tag, all inside the body. You can see the structure of what you made, as well as the structure of this page by opening your browser's web inspector.
