Formatting Help

Our markdown-inspired formatting parser allows you to add formatting to your post such as emphasis and code.

Inline formatting #

Inline formatting consist of simple tags, special tags, and spans.

Simple tags #

Simple tags consist of a markdown character sequence, a text span, then the same sequence. They can be nested in any other tag except themselves. Text within a simple tag is parsed with the exception of code and samp tags.

M↓ HTML
` code
`` samp
* em
** strong
_ i
––– s
+++ ins
--- del
[c] cite
[m] mark
[u] u
[v] var
:: kbd
" q

Example: strong emphasis

**I** prefer green elephants, but **he** prefers pink ones.
I prefer green elephants, but he prefers pink ones.

Escapement #

To write literal formatting characters, for example the literal string "3*4*5 = 60", escape the parsed characters with a backslash (3\*4\*5 = 60). To write a literal backslash, escape it with another backslash.

*emphasis* \*just asterisks\*
emphasis *just asterisks*

Special tags #

Special tags consist of a starting markdown character sequence, a text span, then an ending character sequence. They can be nested in themselves and any other tag. Text within a special tag is parsed.

Start End HTML
^( )^ sup
v( )v sub
[sm] [/sm] small

Example: small text

Free unicorns! [sm]may eat your daisies[/sm].
Free unicorns! may eat your daisies.

Text spans #

Text spans are the bits of text between tags. They have markdown features of their own.

  • Links #

    Bare URLs with http:// or https:// are automatically turned into a link:

    http://example.com
    example.com

    But it is preferable to use a link title:

    [Example title](http://example.com)
    Example title

  • Images #

    Embedded mages have similar syntax as links, just with an !. Alt text is shown when images are disabled or otherwise unavailable.

    ![Alt text](https://devdoodle.net/a/logosquare.svg)

    A width can be optionally specified, enclosed in brackets, after the alt text:

    ![Alt text][12em](https://devdoodle.net/a/logosquare.svg)

  • Abbreviations #

    An expantion of an abbreviation is shown as hover text.

    [HTML|Hyper Text Markup Language] is a markup language used for web pages.
    HTML is a markup language used for web pages.

  • References #

    A reference consists of a number and optional explanatory hover text. The number corresponds to the reference in a reference list, usually placed at the bottom of a post.

    Pigs can't fly[[1]Page 42, start of paragraph 2].
    Pigs can't fly[1].

Block formatting #

Block formatting controls the type of block a line is in. By default, lines are separated into paragraphs.

Blockquote #

Blockquotes can contain anything a post can. Each line in a blockquote starts with > .

> Blockquote > > Blockquote inside a blockquote

Blockquote

Blockquote inside a blockquote

Unordered list #

Unordered lists items are started with a dash (- ) or an asterisk (* ). Additional lines inside a list item must be indented by 4 spaces or a tab character.

* Item 1 - Item 2 Line 2 of item 2
  • Item 1

  • Item 2

    Line 2 of item 2

Ordered list #

Ordered lists are like unordered lists, except that items have a defined order and are usually rendered with numbers prefixed to list items. Items are started with a sequence of one or more digits (0-9) or a single letter (A-Z, case insensitive) followed by either ) or . (period character). Additional lines inside a list item must be indented by 4 spaces or a tab character.

1. Item 1 b. Item 2 Line 2 of item 2 C) Item 3 39. Item 4
  1. Item 1

  2. Item 2

    Line 2 of item 2

  3. Item 3
  4. Item 4

Block code #

Lines indented by 4 spaces or a tab character (or more depending on nesting in lists) are not parsed and are put in a code block.

function multiply3(a, b, c) { return a * b * c; }
function multiply3(a, b, c) { return a * b * c; }

Headers #

6 levels of headers can be created with 1-6 #s followed by a space.

# Level 1 header ## Level 2 header ### Level 3 header

Level 1 header

Level 2 header

Level 3 header

Horizontal rule #

A horizontal rule separates 2 sections of a post. It can be created with 12 or more - (dash) or – (long dash).

Section A. ------------ Section B.

Section A.

Section B.

Reference List #

A reference list, typically at the end of a post, lists the sources and references used in the post. Each numbered item can be refernced as an inline reference. Items usually include the [c] cite element.

All giraffes are giraffes[[1]By an inductive argument described on page 144]. cite[1]: Chapter 2 of [c][example document](http://example.com)[c].

All giraffes are giraffes[1].

1 Chapter 2 of example document.
